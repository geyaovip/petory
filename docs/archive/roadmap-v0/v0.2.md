# V0.2 — 图片上传与 AI 桌宠生成

## 版本目标

证明用户可以上传自己的图片，由 Petory **自动生成**专属桌宠并显示在桌面上。

图片处理是 **AI 桌宠生成**，不是抠图工具；用户只需：上传 → 等待 → 使用。

## 功能范围

### 必须做

1. 欢迎页 + 上传页 + 生成中页 + 结果页 + 命名页
2. 图片格式校验（PNG / JPG / JPEG / WEBP，≤ 10MB）
3. 自动压缩（最长边 1280px）
4. 调用 MiniMax 图生图（Petory Style，3/4 坐姿）
5. 调用 rembg 去背景（后台，不对用户展示）
6. 输出透明 PNG（`pet.png`）
7. 结果页：Use This Pet / Regenerate / Upload Another Photo
8. 宠物命名 + 性格选择
9. 本地保存宠物数据与文件
10. 桌宠窗口切换为用户生成的 `pet.png`

### 不做

- 原图抠图版
- 手动修图、裁剪、擦边
- 姿势选择
- 风格选择（MVP 固定 Petory Style，`styleType` 预留）
- AI 对话
- 番茄钟、成长系统
- 完整设置页

## 生成流水线

```
上传 → 校验 → 压缩 → MiniMax 生成 → rembg → 透明 PNG → 结果页 → 命名 → 桌面展示
```

## 文件存储

```
appData/pets/{petId}/source/original.{ext}
appData/pets/{petId}/source/compressed.{ext}
appData/pets/{petId}/generated/minimax_raw.png
appData/pets/{petId}/generated/pet.png
```

## 开发任务

| # | 任务 |
|---|------|
| 1 | 图片上传 UI + 格式/大小校验 |
| 2 | 主进程图片压缩（max edge 1280px） |
| 3 | 接入 MiniMax 图生图 API + Petory Style Prompt |
| 4 | 接入 rembg（主进程 subprocess） |
| 5 | 实现生成中 loading 页（不暴露 rembg 步骤） |
| 6 | 实现结果页三按钮 |
| 7 | 失败处理（生成失败 / rembg 失败，无原图兜底） |
| 8 | Pet 数据模型 + 本地持久化 |
| 9 | 命名页 + 性格选择 |
| 10 | 桌宠窗口加载 `pet.png` |

## 验收标准

- [ ] 用户可上传合规图片
- [ ] 系统生成 3/4 坐姿、单主体、面向用户的桌宠
- [ ] 最终输出透明 PNG，适合 160–220px 桌面显示
- [ ] 用户可使用 / 重新生成 / 重新上传
- [ ] 不出现原图版、修图工具、姿势/风格选择
- [ ] 关闭重开后宠物仍在，图片与数据不丢失
- [ ] 首次创建流程 < 1 分钟（网络正常时）

## 失败提示

| 场景 | 文案 |
|------|------|
| 上传不合规 | 提示更换格式或缩小至 10MB 以内 |
| 生成失败 | `We couldn't generate a desktop pet from this image. You can try again or upload another photo.` |
| rembg 失败 | `Your pet was generated, but we couldn't prepare it for the desktop. Try regenerating it.` |
