# V1.1+ — 增强能力（后续版本）

## 版本定位

V1.0 公开发布后的迭代路线。以下能力**已在产品规划中保留**，MVP 不实现，但 Schema 与文档不得删除。

## 功能模块

### 多风格生成（V1.1 建议优先级）

恢复风格选择页，用户可在生成前或重新生成时选择风格：

| `styleType` | 名称 | 说明 |
|-------------|------|------|
| `petory` | Petory Style | 默认 Q 版桌宠（MVP 已有） |
| `pixel` | Pixel | 复古像素风 |
| `sticker` | Sticker | 扁平贴纸风 |
| `plush` | Plush | 毛绒玩偶风 |
| `clay` | Clay | 黏土手作风 |
| `cyber` | Cyber Pet | 赛博科幻风 |

- 每种风格独立 MiniMax Prompt 变体
- 可对接 Pro 付费解锁或按次付费
- 姿势仍为 3/4 坐姿（姿势选择在更后版本）

### 多姿势 / 多状态图（V1.2+）

为 idle、happy、sleep、focus 等状态生成或切换不同姿势图，减少纯 CSS 动画依赖。

### 多宠物（V1.1–V1.2）

- 创建多只宠物，同一时间默认只显示一只
- 高级版可同时显示多只

### 体验增强

- 点击穿透模式
- 全屏自动隐藏桌宠
- 语音聊天 / TTS
- 装饰、主题包、动作包
- 社区分享

### 长期方向

- Live2D
- 云端全量同步
- 会员订阅体系完善
- 移动端（非当前重点）

## 开发原则

1. **不破坏 V1.0 本地用户数据** — 迁移脚本必备
2. **`styleType` 等字段从 MVP 已预留** — 直接扩展枚举
3. **新能力默认 Pro 或按次付费** — 与商业化一致
4. **每个子版本独立验收** — 不与大版本混发
