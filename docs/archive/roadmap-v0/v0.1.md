# V0.1 — 内部原型（桌面窗口验证）

## 版本目标

证明 Petory 桌宠能在电脑桌面上以透明窗口常驻运行，具备最基础的互动能力。

**本版本不涉及图片上传与 AI 生成**，使用内置或固定的示例 PNG 验证技术可行性。

## 功能范围

### 必须做

1. Electron 项目初始化（React + TypeScript + Vite）
2. 透明、无边框、默认置顶桌宠窗口
3. 窗口内展示固定 PNG 宠物图
4. 拖拽移动桌宠
5. 保存并恢复窗口位置
6. 点击桌宠显示气泡（5 秒后自动消失）
7. 右键菜单：和它说话（占位）、开始专注（占位）、设置（占位）、隐藏、退出

### 不做

- 图片上传
- MiniMax / rembg
- AI 对话
- 番茄钟、成长系统
- 设置页完整实现
- 双平台打包

## 开发任务

| # | 任务 |
|---|------|
| 1 | 初始化 Electron + React + TS + Vite 项目骨架 |
| 2 | 主进程创建透明置顶 `BrowserWindow` |
| 3 | 渲染进程展示宠物 PNG + CSS 待机动画（上下浮动） |
| 4 | 实现拖拽与位置持久化（localStorage 或 JSON 即可） |
| 5 | 点击气泡组件 |
| 6 | 右键 `contextMenu` |
| 7 | 隐藏 / 退出应用 |

## 验收标准

- [ ] 应用启动后桌宠出现在桌面
- [ ] 窗口无边框、背景透明、默认置顶
- [ ] 可拖拽，重启后位置恢复
- [ ] 点击出现气泡，约 5 秒后消失
- [ ] 右键菜单可弹出，退出可关闭应用
- [ ] 待机动画（轻微浮动）正常

## 技术备注

- 先单平台验证透明窗口渲染（macOS 与 Windows 行为有差异）
- 为 V0.2 预留 IPC 通道命名：`petory:window:*`
