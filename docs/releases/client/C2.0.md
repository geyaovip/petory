## 启用方式

任选一种方式配置后台地址（**设置里填的优先**）：

1. **设置 → 后台服务 → API 地址**（正式包推荐）
2. 开发环境根目录 `.env`：

```env
PETORY_API_BASE_URL=http://localhost:8787
```

### 一键启动

**全 Docker（API + 官网 + 库）+ 本机客户端：**

```bash
npm run dev:docker
```

**传统混合（本机跑 API）：**

```bash
npm run dev:stack
```

详见 [DOCKER-DEV.md](../../development/DOCKER-DEV.md)。Docker 启动后官网在 http://localhost:5180/ ，无需单独 `website:preview`。

### 分步启动

```bash
npm run server:db:up   # 首次：启动 Postgres
npm run server:db      # 同步表结构
npm run server:dev     # 终端 1
npm run dev            # 终端 2
```

数据库说明见 [DATABASE.md](../../backend/DATABASE.md)。

配置 `PETORY_API_BASE_URL` 后：

| 能力 | 行为 |
|------|------|
| 登录/注册 | `POST /api/auth/*`，JWT 存本地 `auth-session.json` |
| 生成 | `POST /api/generation/batch` 等，服务端 MiniMax，客户端 rembg |
| 对话 | `POST /api/chat/send`，Key 不出客户端 |
| 兑换码 | `POST /api/redeem` |
| 额度 | 服务端权威，`GET /api/me` 同步 |
| 维护公告 | 启动拉取 `GET /api/app/status` |

**须登录**：已移除「暂不登录，本地使用」；启动后需注册或登录账号。

未配置后台地址时，仍可使用本地 Mock 账号（仅开发调试）。

## 代码结构

| 路径 | 说明 |
|------|------|
| `electron/main/api/client.ts` | 带 JWT 的 fetch 封装 |
| `electron/main/api/appStatus.ts` | 维护公告与服务开关缓存 |
| `electron/main/api/remoteQuotaStore.ts` | 服务端额度缓存 |
| `electron/main/auth/authService.ts` | Mock / Remote 路由 |
| `electron/main/auth/remoteAuth.ts` | 远程登录、注册、兑换 |
| `electron/main/auth/sessionGuard.ts` | 401 过期处理 |
| `electron/main/image/remoteGeneration.ts` | 上传图片、下载原图 |
| `electron/main/chat/remoteChat.ts` | 远程对话 |
