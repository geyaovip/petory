# C2.3 体验收口

> 版本：**2.1.0** · 阶段 A 内测准备

## 已实现

| 项 | 说明 |
|----|------|
| 维护公告 | 登录页、设置页展示 `maintenanceNotice` |
| 注册开关 | 后台关闭注册时，登录页禁用「注册」Tab |
| 服务状态 | 设置页显示生成/对话维护提示 |
| 额度刷新 | 打开设置、`auth.refresh`、生成/对话成功后更新剩余次数 |
| 401 处理 | Token 过期自动退出并回到登录页 |
| 一键启动 | `npm run dev:stack`（Postgres + API + 客户端） |

## 启动内测环境

根目录 `.env`：

```env
PETORY_API_BASE_URL=http://localhost:8787
```

```bash
npm run dev:stack
```

或分步：

```bash
npm run server:db:up
npm run server:db
npm run server:dev   # 终端 1
npm run dev          # 终端 2
```

验收清单见 [QA-INTERNAL.md](../../quality/QA-INTERNAL.md)。

## API

| IPC | 说明 |
|-----|------|
| `auth.refresh` | 拉取 `/api/app/status` + `/api/me`，刷新额度与公告 |
| `auth.stateChanged` | 额度/会话变化时广播 |

## 产品决策（已定）

| 项 | 决策 |
|----|------|
| 后台 API 地址 | **可配置**：设置 → 后台服务 → API 地址；优先于 `.env` 的 `PETORY_API_BASE_URL` |
| MiniMax / Kimi Key | **设置页不展示**；接后台后由服务端代理，本机无需填 Key |
| 离线模式 | **已移除**；须注册/登录后使用 |
