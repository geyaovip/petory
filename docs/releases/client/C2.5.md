# C2.5 阶段 C — 模拟支付

> 客户端 **2.3.0** · 管理端 **B1.4.0**  
> 真支付（微信/支付宝等）与图片审核**不在本阶段**，后续单独接入。

## 已实现

| 项 | 说明 |
|----|------|
| 订阅方案 | 月付 `pro_monthly`（¥18/30 天）、年付 `pro_yearly`（¥168/365 天，演示价） |
| 模拟支付流程 | 创建订单 → `mock/confirm` 确认 → 开通 Pro、写入 `proExpiresAt` |
| 到期降级 | 登录、`/api/me`、鉴权中间件自动将过期 Pro 降为 free |
| 客户端 UI | 设置 → Pro 升级：选方案 +「模拟支付开通」 |
| 管理端 | 系统配置开关、支付订单列表、用户 Pro 到期日 |
| 兑换码 | 仍可用；兑换 Pro 无到期日（`proExpiresAt = null`） |

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/payment/plans` | 公开，订阅方案列表 |
| POST | `/api/payment/checkout` | 需登录，创建 pending 订单 |
| POST | `/api/payment/mock/confirm` | 需登录，模拟支付成功 |
| GET | `/api/payment/orders` | 需登录，本人订单 |
| GET | `/api/admin/payment-orders` | 管理员查看订单 |

`GET /api/app/status` 新增：

- `paymentEnabled` — 是否展示支付入口
- `mockPaymentEnabled` — 是否允许模拟确认

## IPC

| 通道 | 说明 |
|------|------|
| `payment.getPlans` | 拉取方案 |
| `payment.purchaseMock` | 一键 checkout + mock 确认，刷新额度，可选触发姿势补全 |

## 数据库

新增表 `PaymentOrder`。升级后执行：

```bash
npm run server:db
```

## 验收

1. 设置 → 选择年付 → 模拟支付 → plan 变为 `pro`，显示到期日  
2. 管理后台可见 paid 订单  
3. 后台关闭「模拟支付」后，客户端提示不可用，兑换码仍可用  
4. 将用户 `proExpiresAt` 改为过去时间，下次请求后降为 free
