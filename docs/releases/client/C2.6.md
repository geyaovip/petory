# C2.6 内测补强

> 客户端 **2.4.0** · 管理端 **B1.4**

## 已实现

| 项 | 说明 |
|----|------|
| 设备封禁生效 | 后台标记异常设备后，生成/对话/兑换/支付等 API 返回 `DEVICE_FLAGGED`（403） |
| 完整备份导入 | 导出 schema v3（含 `petFiles`）；设置 → 从备份导入，自动 pre-import 备份 |
| API 冒烟脚本 | `npm run qa:smoke` 检查 health / app/status / payment plans |

## 设备 ID

客户端所有带 JWT 的请求自动附加：

```
X-Petory-Device-Id: <localDeviceId>
```

生成接口仍通过表单字段 `deviceId` 校验（与 `localDeviceId` 一致）。

## 备份格式（schema v3）

```json
{
  "schemaVersion": 3,
  "userDataRoot": "/path/to/userData",
  "pets-store.json": { ... },
  "petFiles": [{ "petId": "...", "relativePath": "generated/idle.png", "data": "<base64>" }]
}
```

- **不导入** `auth-session.json` / `mock-users.json` — 换机后请重新登录
- v2 旧备份可导入 JSON，但**无图片**，路径可能失效
