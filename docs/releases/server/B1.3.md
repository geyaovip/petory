# B1.3 交付说明

> 基础运营能力 **B1.3**；支付与设备封禁见 **B1.4**（[C2.5](../client/C2.5.md)、[C2.6](../client/C2.6.md)）
> 代码目录：`server/`

## 新增能力

| 模块 | API / 能力 | 说明 |
|------|------------|------|
| 公开状态 | `GET /api/app/status` | 维护公告、服务开关、额度上限（供客户端 C2.3 拉取） |
| 系统配置 | `GET/PATCH /api/admin/system/config` | 免费/Pro 生成与对话额度、注册开关、服务开关、维护公告、任务超时 |
| 增强仪表盘 | `GET /api/admin/dashboard` | 7 日趋势、失败原因分布、今日额度不足次数、今日新注册 |
| 审计日志 | `GET /api/admin/audit-logs` | 管理员写操作记录 |
| 登录日志 | `GET /api/admin/login-logs` | 用户登录成功/失败 |
| 设备管理 | `GET /api/admin/devices`、`POST .../flag` | 设备列表与异常标记 |
| 只读运营 | `role=operator` | 可查看全部数据，写入接口返回 403 |
| 配额热更新 | 配置保存后 | 同步更新所有用户 `dailyLimit` |

## 系统配置字段

| 字段 | 默认 | 说明 |
|------|------|------|
| `freeDailyGenerationLimit` | 3 | 免费用户每日生成 |
| `proDailyGenerationLimit` | 50 | Pro 每日生成 |
| `freeDailyChatLimit` | 20 | 免费每日对话 |
| `proDailyChatLimit` | 9999 | Pro 每日对话 |
| `registrationOpen` | true | 是否开放注册 |
| `generationServiceEnabled` | true | 关闭后生成 API 返回维护提示 |
| `chatServiceEnabled` | true | 关闭后对话 API 返回维护提示 |
| `maintenanceNotice` | 空 | 客户端可展示的公告文案 |
| `jobTimeoutMs` | 60000 | 生成任务超时 |

## 公开状态示例

```json
GET /api/app/status
{
  "version": "B1.3.0",
  "registrationOpen": true,
  "generationServiceEnabled": true,
  "chatServiceEnabled": true,
  "maintenanceNotice": null,
  "limits": {
    "free": { "dailyGenerationLimit": 3, "dailyChatLimit": 20 },
    "pro": { "dailyGenerationLimit": 50, "dailyChatLimit": 9999 }
  }
}
```

## 账号

| 角色 | 默认邮箱 | 默认密码 | 权限 |
|------|----------|----------|------|
| 管理员 | `admin@petory.app` | `petory-admin` | 读写 |
| 运营（只读） | `operator@petory.app` | `petory-operator` | 仅查看 |

可通过环境变量覆盖：`OPERATOR_EMAIL`、`OPERATOR_PASSWORD`。

## 数据库与迁移

管理端使用 **PostgreSQL**（见 [DATABASE.md](../../backend/DATABASE.md)）。

```bash
cd server
npm run db:up      # 本地 Docker Postgres
npm run db:push
npm run dev
```

首次启动会自动写入 `SystemConfig` 默认值，并种子化运营账号（若不存在）。

## 验收

- [ ] `PATCH /api/admin/system/config` 修改额度后，`GET /api/generation/quota` 反映新上限
- [ ] 关闭 `generationServiceEnabled` 后 batch 返回 `SERVICE_DISABLED`
- [ ] 关闭 `registrationOpen` 后注册返回明确错误
- [ ] 额度用尽时 `QuotaLog` 有 `changeType=blocked` 记录
- [ ] 仪表盘显示 7 日趋势与失败分布
- [ ] 运营账号无法创建兑换码或禁用用户
- [ ] `GET /api/app/status` 无需登录可访问

## 客户端

已对接（C2.0+）。客户端拉取 `/api/app/status`，并在请求头携带 `X-Petory-Device-Id` 以配合设备封禁（C2.6）。
