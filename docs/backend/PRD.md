# Petory 服务端 PRD

更新日期：2026-06-17。

## 目标

为 Petory 客户端提供真实登录、设备管理、六姿势宠物生成、聊天、统一额度和运营可观测性。

## 登录

- 用户和管理员均使用邮箱 Magic Link，不提供密码登录。
- 用户 Magic Link 与管理员 Magic Link 使用不同 token 表，不能交叉验证。
- 管理员只允许 `ADMIN_EMAIL` 配置的账号；运营账号可配置为只读。
- 登录页面不预填邮箱，公开代码与文档不保存真实管理员邮箱。

## 宠物生成

- 不提供风格选择，始终保持上传主体身份。
- **每个账号终身仅可成功创建 1 只自定义宠物**；成功后不可重新生成；删除本地记录不重置服务端名额。
- 每只宠物默认生成待机、开心、专注、睡觉、提醒、生气 6 种姿势。
- 图像生成使用火山引擎 Doubao-Seedream-4.5（`doubao-seedream-4-5-251128`）。
- 服务端生成任务保存输入与输出路径。
- 客户端本地生成完成后上传 360px WebP 预览，用于管理端排查任务。
- 历史本地任务没有预览时，管理端明确说明数据来源，不伪装成加载失败。

## 管理端

- 数据概览、用户状态与额度、设备、生成批次与任务、聊天与审计日志、系统配置。
- 写操作仅管理员可用，运营账号只读。
- 会员、支付和兑换能力当前停用，管理端和公开 API 均不暴露入口。

## 兼容性

数据库暂时保留旧版计划、支付和兑换表，避免破坏历史数据。当前业务逻辑统一按普通用户规则执行；重新启用前必须另立需求和迁移方案。

## 验收

- Magic Link 可真实发送、一次性消费并区分用户与管理员权限。
- 所有新宠物默认请求 6 姿势。
- 已创建过自定义宠物的账号，再次创建或重新生成 API 返回 `CUSTOM_PET_LIMIT`（409），账号其他能力不受影响。
- 新版客户端本地生成记录在管理端可见预览。
- 登录输入框、示例环境和公开文档不含真实管理员邮箱。
- 管理端没有会员、支付或兑换入口。
