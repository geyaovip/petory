# 客户端与服务端对齐

更新日期：2026-06-17。

| 能力 | 客户端 | 服务端 |
|---|---|---|
| 登录 | 邮箱 Magic Link | 一次性 token 验证并签发 JWT |
| 管理员登录 | 不适用 | 独立管理员 Magic Link，仅允许配置邮箱 |
| 宠物生成 | 固定 Petory 身份保持风格 | 固定 6 姿势，服务端校验姿势 |
| 自定义宠物名额 | 每账号 1 只；隐藏新建/重新生成入口 | `customPetCreatedAt` + 生成 API 拦截（非封禁账号） |
| 本地生成记录 | 上传 360px WebP 姿势预览 | 保存预览并在管理端展示 |
| 生成额度 | 展示统一额度 | 所有用户使用同一套额度 |
| 聊天额度 | 展示统一额度 | 所有用户使用同一套额度 |
| 会员与支付 | 不展示 | 路由停用，旧数据字段仅兼容历史记录 |

## 姿势

`idle`、`happy`、`focus`、`sleep`、`remind`、`angry` 均默认生成，不再按用户计划区分。

## 隐私

- 客户端与管理端登录输入框必须为空，不得预填真实邮箱。
- 管理员邮箱仅配置在未提交的生产环境变量中。
- 示例环境文件和公开文档不得出现真实个人邮箱或 API Key。
