# Petory 内部 QA 清单

更新日期：2026-06-17。

## 基础验证

- [ ] `npm run typecheck`
- [ ] `npm run build`
- [ ] `npm --prefix server run db:generate`
- [ ] 启动 API 后访问 `/health`
- [ ] `npm run qa:brand`

## 登录与隐私

- [ ] 客户端登录输入框初始为空。
- [ ] 管理端登录输入框初始为空，退出后清空。
- [ ] 用户与管理员 Magic Link 均可真实发送并登录。
- [ ] 用户 token 不能登录管理端，管理员 token 不能登录客户端。
- [ ] 普通用户不能进入管理端。
- [ ] Git 暂存内容不包含真实管理员邮箱、Resend Key 或生产 `.env`。

## 宠物生成

- [ ] 新建宠物不显示风格选择。
- [ ] 默认生成待机、开心、专注、睡觉、提醒、生气 6 种姿势。
- [ ] 六张图保持同一主体身份，无卡片、边框、文字和额外道具。
- [ ] 默认示例宠物六姿势均为透明背景半写实假 3D 资源。
- [ ] 姿势切换动画与提醒、专注、睡觉状态匹配。
- [ ] 已创建自定义宠物的账号无法再次新建或重新生成；账号仍可登录、聊天、使用已有宠物。
- [ ] 生成失败且未产出成功批次时，账号不应被误占自定义宠物名额。
- [ ] 云端恢复导入仍可拉取该账号历史成功批次。

## 管理端

- [ ] 不显示会员、支付、兑换码入口或指标。
- [ ] 用户列表可管理状态与赠送统一额度。
- [ ] 服务端生成任务显示输出预览。
- [ ] 新版客户端本地生成任务显示压缩预览。
- [ ] 旧版客户端无图任务显示“旧版客户端未上传预览”。
- [ ] 运营账号不可执行写操作。

## 部署

- [ ] 生产 `.env` 配置 `ADMIN_EMAIL`、`RESEND_API_KEY`、`MAIL_FROM` 和强随机 `JWT_SECRET`。
- [ ] `prisma db push` 成功。
- [ ] `/health` 返回成功。
- [ ] `/admin/` 可访问且管理端 Magic Link 可完成登录。
