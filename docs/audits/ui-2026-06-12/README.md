# UI 审查：2026-06-12

## 范围

- Electron 客户端：登录/注册、创建桌宠全流程、聊天、专注、成长、设置、宠物管理、功能指南、桌宠浮层。
- 用户官网：首页、下载、隐私政策、用户协议，桌面与 390px 移动视口。
- 管理端：登录、概览、系统配置、用户、设备、支付与兑换、生成记录、对话与审计，桌面与窄屏结构。

## 方法

1. 逐页阅读 JSX、HTML、CSS 与交互脚本，确认页面状态和共享组件。
2. 在本地预览官网与管理端，检查 1440px 桌面和 390px 移动视口。
3. 对客户端使用实际 Electron 窗口与源码状态矩阵交叉检查；本地 API 不可用时不提交账号或生成请求。
4. 优先修复能够覆盖多个页面的组件、布局和交互契约，再修正用户端高频流程。

## 主要问题与处理

| 优先级 | 问题 | 处理 |
|--------|------|------|
| P0 | 官网移动端直接隐藏导航，用户无法稳定到达隐私、协议等页面 | 增加可键盘操作的移动菜单、当前页状态和焦点样式 |
| P0 | 客户端上传区用可点击 div，键盘无法触发 | 改为语义化 button，补齐 disabled、busy 和 focus 状态 |
| P1 | 客户端按钮、选择器、文字操作的焦点反馈不统一 | 在共享 UI 组件统一 focus-visible、disabled 与选中语义 |
| P1 | 确认弹窗缺少 Escape 和初始焦点 | 增加键盘关闭并将焦点移入弹窗 |
| P1 | 页面壳在最小窗口中可能裁掉长内容 | 内容区改为独立滚动，保持标题栏与窗口结构稳定 |
| P1 | 欢迎、加载、空态使用无意义圆块或临时绘制装饰 | 统一使用真实 Petory 品牌资产 |
| P1 | 管理端窄屏把全部导航横向折行，信息层级崩坏 | 改为抽屉式侧栏，顶部提供菜单按钮 |
| P1 | 管理端高风险操作缺少明确确认，写操作可重复触发 | 禁用用户/兑换码状态变更加确认，操作期间锁定按钮并标记 busy |
| P2 | 官网功能卡完全同权，阅读路径弱 | 增加序号与功能类别，收紧移动端留白和卡片间距 |
| P2 | 三端缺少统一的 reduced-motion 规则 | 客户端、官网、管理端补充减少动态效果规则 |

## 仍需真实环境复核

- 管理端登录后的真实数据密度、超长邮箱/错误原因和大表格横向滚动。
- 客户端真实上传、远端生成、额度耗尽、维护模式与会话过期流程。
- macOS 与 Windows 最小窗口下的系统标题栏差异。

这些项目已写入 [UI 设计规范](../../design/UI-GUIDELINES.md) 的页面验收矩阵，后续版本按矩阵回归。
